1. Verify the controller is running:

   kubectl -n {{ .Release.Namespace }} rollout status deploy/{{ include "kopiur.controller.fullname" . }}

   kubectl -n {{ .Release.Namespace }} rollout status deploy/{{ include "kopiur.webhook.fullname" . }}
{{- end }}

2. Confirm the CRDs are registered:

   kubectl get crd -l app.kubernetes.io/part-of=kopiur
   # repositories, clusterrepositories, snapshotpolicies, snapshots,
   # snapshotschedules, restores, maintenances, repositoryreplications
   # (all in group kopiur.home-operations.com)

{{- if .Values.webhook.enabled }}
{{- if eq .Values.webhook.tls.mode "self" }}

   Webhook TLS: self-managed (webhook.tls.mode: self). The controller mints the
   serving cert into Secret "{{ .Values.webhook.tls.secretName }}" and injects the caBundle into
   the webhook configurations — no cert-manager and no manual steps. The webhook
   pod stays in ContainerCreating until the controller creates that Secret (a few
   seconds after the controller becomes ready).
{{- else if eq .Values.webhook.tls.mode "cert-manager" }}

   Webhook TLS: cert-manager (webhook.tls.mode: cert-manager). cert-manager must
   be installed; it issues the serving cert and its ca-injector populates caBundle.
{{- else if eq .Values.webhook.tls.mode "manual" }}

   NOTE: webhook.tls.mode is "manual". You must:
     a) create the TLS Secret "{{ .Values.webhook.tls.secretName }}" (kubernetes.io/tls) in
        namespace {{ .Release.Namespace }} with the webhook serving cert, and
     b) set webhook.caBundle (base64 PEM) so the API server trusts it.
   Otherwise admission of kopiur.home-operations.com objects will fail{{ if eq .Values.webhook.failurePolicy "Fail" }} (failurePolicy=Fail){{ end }}.
{{- end }}
{{- end }}

3. Create your first repository and start backing up a PVC. Minimal example:

   apiVersion: v1
   kind: Secret
   metadata: { name: nas-creds, namespace: {{ .Release.Namespace }} }
   stringData:
     AWS_ACCESS_KEY_ID: "..."
     AWS_SECRET_ACCESS_KEY: "..."
     KOPIA_PASSWORD: "choose-something-long"
   ---
   apiVersion: kopiur.home-operations.com/v1alpha1
   kind: Repository
   metadata: { name: nas-primary, namespace: {{ .Release.Namespace }} }
   spec:
     backend:
       s3:
         bucket: my-backups
         endpoint: s3.us-east-1.amazonaws.com
         region: us-east-1
         auth: { secretRef: { name: nas-creds } }
     encryption: { passwordSecretRef: { name: nas-creds, key: KOPIA_PASSWORD } }
     create: { enabled: true }

   See deploy/examples/ for runnable walkthroughs (scheduled backup,
   ClusterRepository, restore, deploy-or-restore GitOps, maintenance,
   repository replication, ...).

4. Watch your resources:

   kubectl get repositories,snapshotpolicies,snapshotschedules,snapshots,restores -A
   kubectl get maintenances,repositoryreplications -A
